# HERMES MASTER HANDOFF TEMPLATE

> Шаблон для передачи задач между Hermes-сессиями.
> Используй как структуру при любых handoff-задачах.

---

## 1. MODE

| Режим | Описание |
|-------|----------|
| **READ-ONLY** | Только чтение. Ничего не менять. Анализ и отчёт. |
| **HARDENING** | Укрепление существующего кода. Тесты, фиксы, документация. |
| **APPLY** | Применение изменений. Новые файлы, фичи, рефакторинг. |

Укажи один режим перед началом работы.

---

## 2. SOURCE OF TRUTH

Обязательно прочитать перед началом:

1. `HERMES.md` — идентичность и навыки агента
2. `AGENTS.md` — контекст проекта и правила
3. `docs/agent-guidelines.md` — operational guidelines
4. Целевые файлы задачи (указать конкретные пути)

Не начинай работу без чтения первых трёх файлов.

---

## 3. TASK

### Цель
Одна точная цель. Одно предложение.

### Scope
- Что входит в задачу
- Какие файлы/папки затрагиваются

### Разрешено
- Список действий, которые можно выполнять

### Запрещено
- Список действий, которые нельзя выполнять
- Файлы, которые нельзя трогать

---

## 4. EXECUTION RULES

1. **Один шаг → один результат → одна проверка**
2. Не выдумывать то, что уже есть в GitHub
3. Не трогать ядро (HERMES.md, AGENTS.md) без разрешения
4. Если данных не хватает — остановиться и указать, чего именно не хватает
5. Не имитировать выполнение — только реальные действия
6. Не менять файлы за пределами scope задачи

---

## 5. OUTPUT FORMAT

### A. VERDICT
ПРИНИМАЕМ / НЕ ПРИНИМАЕМ

### B. WHAT WAS READ
- Список реально прочитанных файлов

### C. WHAT WAS FOUND
- Ключевые находки по задаче

### D. WHAT WAS DONE
- Конкретные изменения (файлы, строки, коммиты)

### E. RISKS
- Возможные проблемы или side effects

### F. NEXT SINGLE STEP
- Только один следующий шаг

---

## 6. CANONICAL SHORT TEMPLATE

```
## HANDOFF: [название задачи]

**MODE:** READ-ONLY / HARDENING / APPLY
**DATE:** YYYY-MM-DD
**AGENT:** HERMES

### SOURCE OF TRUTH
- [ ] HERMES.md
- [ ] AGENTS.md
- [ ] docs/agent-guidelines.md
- [ ] [целевые файлы]

### TASK
Цель: [одно предложение]
Scope: [файлы/папки]
Разрешено: [действия]
Запрещено: [действия]

### EXECUTION LOG
1. [шаг] → [результат] → [проверка]
2. [шаг] → [результат] → [проверка]

### OUTPUT
- VERDICT: ПРИНИМАЕМ / НЕ ПРИНИМАЕМ
- WHAT WAS READ: [список]
- WHAT WAS FOUND: [находки]
- WHAT WAS DONE: [изменения]
- RISKS: [риски]
- NEXT SINGLE STEP: [следующий шаг]
```
