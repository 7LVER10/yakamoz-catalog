## When to Use

- Клиент запрашивает создание сайта / лендинга
- Получен бриф через форму, Telegram, email
- Нужно проверить, достаточно ли данных для старта

## Prerequisites

- Исходный запрос клиента доступен
- Известен контакт клиента

## Workflow

### 1. Получить исходный запрос
Зафиксировать:
- Контакт клиента (имя / компания)
- Тип запроса (сайт / лендинг / обновление)
- Текст брифа / сообщения

### 2. Выделить базовые параметры

| Параметр | Пример |
|----------|--------|
| **Тип сайта** | Корпоративный / каталог / лендинг / интернет-магазин |
| **Цель** | Продажи / заявки / информирование / брендинг |
| **Целевая аудитория** | B2B / B2C / 특정 сегмент |
| **Страницы / блоки** | Главная / каталог / контакты / о нас |
| **Контент / материалы** | Тексты / фото / логотип / сертификаты |
| **Язык(и)** | TR / EN / RU / AR |
| **Стиль / референсы** | Ссылки на примеры / цвета / шрифты |
| **CTA / интеграции** | Форма заявки / запись / оплата / CRM |

### 3. Проверить полноту брифа

| Параметр | Обязательный? | Есть? |
|----------|---------------|-------|
| Тип сайта | Да | ✓ / ✗ |
| Цель | Да | ✓ / ✗ |
| Целевая аудитория | Да | ✓ / ✗ |
| Контент | Нет (можно сгенерировать) | ✓ / ✗ |
| Язык(и) | Да | ✓ / ✗ |
| Стиль | Нет (есть defaults) | ✓ / ✗ |
| CTA / интеграции | Нет | ✓ / ✗ |

### 4. Отметить пробелы

Если чего-то не хватает:
- Указать конкретный параметр
- Предложить варианты заполнения
- Не гадать — фиксировать неизвестность

### 5. Сформировать intake summary

Структура:
```
## Intake: [название проекта]

**Клиент:** [имя / компания]
**Контакт:** [связь]
**Тип:** [сайт / лендинг]
**Цель:** [одно предложение]
**ЦА:** [описание]
**Страницы:** [список]
**Языки:** [список]
**CTA:** [тип]
**Интеграции:** [список]
**Стиль:** [описание / ссылки]
**Пробелы:** [что не хватает]
```

### 6. Определить следующий шаг

| Пробел | Следующий шаг |
|--------|---------------|
| Нет цели | Уточнить у клиента |
| Нет контента | Предложить генерацию или попросить материалы |
| Нет стиля | Предложить варианты из design-system |
| Всё есть | Передать на проектирование |

### 7. Зафиксировать результат

Запись содержит:
- Дата
- Клиент
- Тип проекта
- Intake summary
- Пробелы
- Следующий шаг

## Verification

- Базовые параметры выделены
- Полнота проверена
- Пробелы зафиксированы
- Intake summary сформирован
- Следующий шаг определён

## Done Criteria

Intake summary содержит: клиент, тип, цель, ЦА, страницы, языки, CTA, пробелы, следующий шаг.

## Limitations

Skill **НЕ** делает:
- Не проектирует весь сайт
- Не генерирует финальный дизайн
- Не пишет полный copywriting
- Не выбирает стек окончательно
- Не обещает сроки или стоимость без подтверждения
