## When to Use

- Раз в неделю при проверке неоплаченных счетов
- По запросу: "проверь статус счёта №..."
- При приближении даты оплаты (за 3 дня до дедлайна)

## Prerequisites

- Данные по счёту доступны (номер, сумма, дата, клиент)
- Известен статус оплаты (оплачен / не оплачен / частично)

## Workflow

### 1. Получить данные по счёту
Зафиксировать:
- Номер счёта
- Сумма
- Дата выставления
- Дата оплаты (дедлайн)
- Клиент (имя / компания / контакт)
- Текущий статус

### 2. Проверить дату и статус
Рассчитать:
- Сколько дней до дедлайна
- Сколько дней просрочки (если есть)
- Текущий статус: оплачен / не оплачен / частично

### 3. Определить просрочку

| Статус | Условие | Действие |
|--------|---------|----------|
| **Оплачен** | Деньги получены | Завершить, зафиксировать |
| **До дедлайна** | > 3 дней | Напомнить клиенту за 3 дня |
| **Дедлайн сегодня** | 0 дней | Срочно связаться |
| **Просрочен** | < 0 дней | Follow-up + эскалация |

### 4. Определить, нужен ли follow-up

| Просрочка | Follow-up |
|-----------|-----------|
| 1–3 дня | Мягкое напоминание |
| 4–7 дней | Повторное напоминание + вопрос |
| 8–14 дней | Эскалация ответственному |
| > 14 дней | Обсуждение взыскания |

### 5. Сформулировать следующий шаг

Конкретное действие:
- "Отправить напоминание клиенту [имя] по [контакт]"
- "Позвонить клиенту [имя] и уточнить статус"
- "Эскалация ответственному: [имя], просрочка X дней"

### 6. Зафиксировать результат

Запись содержит:
- Дата проверки
- Номер счёта
- Сумма
- Статус (оплачен / просрочен на X дней)
- Follow-up: да / нет
- Следующий шаг
- Ответственный

## Verification

- Статус счёта проверен
- Просрочка рассчитана
- Follow-up определён (да / нет)
- Следующий шаг сформулирован

## Done Criteria

Запись содержит: дата, номер, сумма, статус, follow-up, следующий шаг, ответственный.

## Limitations

Skill **НЕ** делает:
- Не отправляет сообщения автоматически
- Не меняет бухгалтерские записи
- Не обещает клиенту ничего от имени компании
- Не блокирует и не разблокирует счета
- Не проводит деньги
