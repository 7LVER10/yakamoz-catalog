## When to Use

При поступлении входящего сообщения от SMB-клиента:
- из Telegram
- из WhatsApp
- по email
- через форму на сайте

## Prerequisites

- Текст сообщения доступен
- Источник сообщения известен

## Workflow

### 1. Получить входящее сообщение
Зафиксировать:
- Текст сообщения
- Источник (Telegram / WhatsApp / email / сайт)
- Имя отправителя (если есть)
- Контакт (если есть)

### 2. Определить источник

| Источник | Приоритет обработки |
|----------|---------------------|
| Telegram | Быстрый (ожидание ответа < 1ч) |
| WhatsApp | Быстрый (ожидание ответа < 1ч) |
| Email | Средний (ответ в течение 24ч) |
| Сайт (форма) | Средний (ответ в течение 24ч) |

### 3. Определить тип запроса

| Тип | Триггер | Пример |
|-----|---------|--------|
| **Новый лид** | Заказ / покупка / интерес | "Хочу купить стекло" |
| **Вопрос по оплате** | Счёт / оплата / долг | "Оплатил счёт, но не видно" |
| **Вопрос по записи** | Встреча / визит / замер | "Хочу записаться на замер" |
| **Support** | Проблема / жалоба / гарантия | "Стекло треснуло" |
| **Прочее** | Не определено | "Привет, как дела?" |

### 4. Проверить полноту данных

Если данных不够 для маршрутизации:
- Текст сообщения есть → продолжить
- Тип определён → продолжить
- Тип не определён → **Остановиться**, запросить уточнение

### 5. Назначить маршрут

| Тип запроса | Downstream-skill | Действие |
|-------------|------------------|----------|
| Новый лид | `lead-triage` | Классифицировать лид |
| Вопрос по оплате | `invoice-chase` | Проверить статус счёта |
| Вопрос по записи | `appointment-followup` | Проверить расписание |
| Support | `manual review` | Направить человеку |
| Прочее | `manual review` | Направить человеку |

### 6. Зафиксировать результат

Запись содержит:
- Дата
- Источник
- Тип запроса
- Выбранный маршрут
- Downstream-skill

### 7. Вернуть routing result

Агент получает:
- Маршрут (название downstream-skill)
- Краткое обоснование
- При необходимости: запрос на уточнение

## Verification

- Источник определён
- Тип запроса определён (один из 5 вариантов)
- Downstream-skill выбран
- Результат зафиксирован

## Done Criteria

Запись содержит: дата, источник, тип, маршрут, downstream-skill.

## Limitations

Skill **НЕ** делает:
- Не отвечает клиенту автоматически
- Не меняет CRM
- Не отправляет сообщения
- Не выполняет downstream-skill полностью
- Не принимает бизнес-решения за человека при недостатке данных
