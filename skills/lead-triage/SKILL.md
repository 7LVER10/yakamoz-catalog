## When to Use

При поступлении нового лида / заявки от клиента:
- через сайт (форма обратной связи)
- через Telegram
- через WhatsApp
- через email

## Prerequisites

- Данные лида доступны (имя, контакт, сообщение)
- Есть доступ к справочнику типов проектов YAKAMOZ

## Workflow

### 1. Получить входные данные
Зафиксировать:
- Имя / компания
- Контакт (телефон / email / Telegram)
- Источник (откуда пришёл)
- Текст сообщения

### 2. Проверить полноту
Если данных不够:
- Имя → продолжить
- Контакт → продолжить
- Описание задачи → **Остановиться**, запросить недостающее

### 3. Определить тип лида

| Тип | Пример |
|-----|--------|
| **Заказ** | "Хочу купить стекло для душевой кабины" |
| **Вопрос** | "Какая толщина стекла нужна?" |
| **Сравнение** | "Сколько стоит vs конкуренты?" |
| **Жалоба** | "Стекло треснуло после установки" |
| **Партнёрство** | "Хочу стать дистрибьютором" |

### 4. Оценить приоритет

| Приоритет | Условие |
|-----------|---------|
| **HIGH** | Жалоба + деньги + давление |
| **MEDIUM** | Заказ с ясным описанием |
| **LOW** | Вопрос / сравнение / общее |

### 5. Определить следующий шаг

| Тип | Приоритет | Следующий шаг |
|-----|-----------|---------------|
| Заказ | MEDIUM | Отправить КП / связаться в течение 24ч |
| Вопрос | LOW | Ответить в течение 48ч |
| Жалоба | HIGH | Немедленно связаться, зафиксировать инцидент |
| Партнёрство | LOW | Направить на email / договор |

### 6. Зафиксировать результат

Создать запись с полями:
- Дата
- Имя / контакт
- Тип лида
- Приоритет
- Следующий шаг
- Кто ответственен

## Verification

- Тип лида определён (один из 5 вариантов)
- Приоритет назначен (HIGH / MEDIUM / LOW)
- Следующий шаг сформулирован
- Данные зафиксированы

## Done Criteria

Запись содержит: дата, контакт, тип, приоритет, следующий шаг, ответственный.

## Limitations

Skill **НЕ** делает:
- Не отвечает клиенту
- Не создаёт коммерческое предложение
- Не меняет статусы в CRM
- Не удаляет и не редактирует существующие записи
